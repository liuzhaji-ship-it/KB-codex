# 项目需求基线：慢阻肺（COPD）医学知识库与智能问答系统

## 项目名称
慢阻肺（COPD）医学知识库与智能问答系统

## 一、项目目标
构建面向 COPD 医学知识的智能知识库系统，支持资料统一管理、可追溯检索与结构化问答。

核心要求：
1. 回答基于知识库内容
2. 回答可追溯来源
3. 回答结构清晰、可信度高

## 二、用户角色
### 1) 医生用户
关注：定义、诊断标准、治疗方案、药物、指南、研究结论、病例经验。

### 2) 患者用户
关注：疾病基础、症状解释、检查意义、治疗方式、生活建议、风险提示。

## 三、知识管理需求
1. 上传资料：PDF / Word / 文本
2. 文档管理：查看、删除、更新、分类、标签
3. 知识更新：持续增量纳入新资料

## 四、查询与问答需求
1. 关键词查询
2. 自然语言提问
3. 主题浏览（定义/诊断/治疗/药物/预防）
4. 多文档综合回答

回答结构要求：
- 结论
- 依据
- 来源（文档名 + 位置）

缺证据时必须返回：
“当前知识库中暂无相关依据。”

## 五、PoC 验证范围（首要）
1. 可上传并纳入知识库
2. 可检索知识片段
3. 可自然语言问答
4. 可返回带引用来源答案

## 六、自动化测试机制（PoC 固定）
1. 直接使用项目目录现有 3 个 PDF 作为初始语料
2. 建立索引并供问答调用
3. 由 Codex CLI 读取 3 PDF 内容并自动生成测试题
4. 题型覆盖：
   - 基础知识
   - 概念解释
   - 文档事实抽取
   - 多文档综合
   - 推理题
   - 引用定位
   - 证据不足判断
5. 将题目提交给知识库系统作答
6. Codex CLI 基于 3 PDF 真实内容自动评估
7. 输出：
   - eval/eval_report.md
   - eval/improvement_todo.md
8. OpenClaw 根据评估结果迭代修复

## 七、评估维度
- 是否基于 3 PDF 内容
- 来源文档名是否正确
- 引用位置是否正确
- 是否存在幻觉
- 证据不足时是否正确拒答
- 回答结构是否清晰
- 多文档综合能力

## 八、系统核心原则
1. 回答必须有依据
2. 回答必须可追溯
3. 回答必须清晰可信
4. 知识库可持续扩展
5. 开发流程必须支持自动化测试与自动评估
