# 底座选择说明（R2）

## 选择结果
- 主底座：**Dify**（用于 PoC 快速闭环）
- 执行底座：**FastAPI 代码执行层**（用于严格测试、可控输出与自动化压测）

## 选择原因
1. Dify 能快速完成“上传文档 -> 检索问答 -> 工作流联动”
2. FastAPI 层便于实现结构化输出、严格拒答、自动评测接口
3. 双层结构可兼顾“快”与“可控”

## 解决的问题
- 快速完成 COPD PoC 演示链路
- 建立可追溯引用问答能力
- 支持 Codex CLI 自动出题与评测回写

## 仍缺失能力
- 医学术语患者友好解释策略（医生/患者双视角模板）
- 更强的引用定位精度（页码/段落稳定定位）
- 多文档冲突证据的优先级规则

## 未来可扩展点
- 向量检索替换（pgvector / Milvus / Qdrant）
- 重排器替换（bge-reranker 等）
- 审查器升级（更严格 hallucination 检测）
