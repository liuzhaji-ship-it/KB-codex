# COPD 知识库系统调研报告（R2）

## 调研目标
为 PoC 选择可商用、可扩展、可快速落地的开源底座，用于实现：3 PDF 建库、可追溯问答、自动出题评测。

## 候选方案对比

### 1) Dify
- 许可：开源（可商用，需遵守其许可证条款）
- 优势：
  - 内置知识库、工作流、RAG 应用编排
  - 上手快，适合 PoC 快速验证
  - 对接模型与向量库配置方便
- 劣势：
  - 深度定制时需要理解其内部链路
  - 对“严格可控输出 schema”需要额外约束
- 部署复杂度：中
- 二开难度：中

### 2) Haystack
- 许可：Apache-2.0（可商用）
- 优势：
  - 模块化强，检索/重排/生成链路清晰
  - 适合严谨工程化与可测试管线
  - 组件替换成本低
- 劣势：
  - 初始开发工作量大于 Dify
- 部署复杂度：中
- 二开难度：中偏高

### 3) LlamaIndex
- 许可：MIT（可商用）
- 优势：
  - 文档摄取与索引能力强，生态成熟
  - 快速搭建多文档问答流程
- 劣势：
  - 复杂生产级治理需要额外工程配套
- 部署复杂度：低到中
- 二开难度：中

### 4) LangChain
- 许可：MIT（可商用）
- 优势：
  - 生态广，组件丰富
  - 与多模型、多向量库兼容好
- 劣势：
  - 若无严格规范，项目易变“拼装链”
- 部署复杂度：中
- 二开难度：中

### 5) RAGFlow
- 许可：开源（需核对具体版本许可证）
- 优势：
  - 面向文档RAG的一体化平台
  - 对文档处理与问答链路有较多现成能力
- 劣势：
  - 项目可控性和定制度需结合场景验证
- 部署复杂度：中
- 二开难度：中

## 结论（调研阶段）
- PoC 快速落地优先：Dify / LlamaIndex
- 工程化可控优先：Haystack
- 当前推荐路线：
  - **PoC 主线：Dify（快速闭环）**
  - **代码可控备线：轻量 FastAPI + 自建RAG链（已在历史仓形成基础）**
